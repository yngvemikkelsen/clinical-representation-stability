# Representation Stability in Clinical Language Models

This repository contains code and data for analyzing representation stability across clinical text complexity levels in transformer-based language models.

## Paper

**Title:** Complexity-Associated Representation Changes in Clinical Language Models: A Four-Group Validation Study with Length Controls

**Authors:** Yngve Mikkelsen, MD, MSc, DBA

**Status:** In preparation for submission to PLOS ONE

## Overview

Large language models are increasingly deployed in clinical settings, yet the stability of their internal representations across varying input complexity remains poorly characterized. This study analyzes representation changes in 8 transformer models (110M-7B parameters, 2018-2023) using both real clinical text and synthetic matched data, with length-controlled comparisons to separate complexity effects from sequence length effects.

### Key Findings

- **6 of 8 models** show magnitude changes >3% from simple to complex clinical text
- **Modern architectures** (Llama-2, Mistral, both 2023) show minimal changes (<1%)
- **Findings replicate** across real and synthetic text (r=0.82-1.00 for most models)
- **Dual metrics** (magnitude + cosine similarity) distinguish geometric changes from scaling artifacts

## Datasets

### Group 1: Real Clinical Text (MT Samples)
- **Source:** MT Samples corpus (mtsamples.com)
- **Size:** 450 clinical notes
- **Stratification:** Simple (6.7±1.8 words), Moderate (21.9±3.2 words), Complex (65.8±7.6 words)
- **Files:** `mt_samples_simple.csv`, `mt_samples_moderate.csv`, `mt_samples_complex.csv`

### Group 2: Synthetic Matched Text
- **Source:** Generated by physician author
- **Size:** 450 texts matched for complexity and length
- **Files:** `synthetic_simple.csv`, `synthetic_moderate.csv`, `synthetic_complex.csv`

### Group 3: Length-Controlled Synthetic
- **Purpose:** Isolate complexity effects from length
- **Size:** 300 texts with matched length but varying complexity
- **Files:** `lengthcontrol_synth_moderate_simple.csv`, `lengthcontrol_synth_complex_simple.csv`

### Group 4: Length-Controlled Real
- **Purpose:** Validate length controls with real clinical text
- **Size:** 300 texts from MT Samples with padding
- **Files:** `lengthcontrol_real_moderate_simple.csv`, `lengthcontrol_real_complex_simple.csv`

**Total:** 1,500 clinical texts across 4 experimental groups

## Metrics

### 1. L2 Magnitude
Mean L2 norm of token embeddings from final hidden layer. Measures representation magnitude changes.

### 2. Cosine Similarity
Mean pairwise cosine similarity between token embeddings within each text. Measures directional changes.

**Dual-metric approach** distinguishes:
- **Geometric changes:** Both magnitude and direction shift
- **Scaling artifacts:** Only magnitude shifts, direction stable
- **Directional changes:** Only direction shifts, magnitude stable

## FAQs

**Q: Can I run this without a GPU?**  
A: Not recommended. 7B models require significant memory and will be extremely slow on CPU.

**Q: Can I use a smaller GPU?**  
A: Minimum recommended is A100 40GB. Smaller GPUs will require 8-bit quantization (see code comments).

**Q: How do I get HuggingFace access to Llama-2 and Mistral?**  
A: Visit the model pages, click "Agree and access repository," and wait for approval (usually instant).

**Q: Can I test additional models?**  
A: Yes! Edit the `MODELS` list in `analyze_representations.py` to add your model.

**Q: What if I only want to test some models?**  
A: Comment out unwanted models in the `MODELS` list.

## Known Issues

- Mistral-7B may require additional memory on A100 40GB (use 8-bit quantization if needed)
